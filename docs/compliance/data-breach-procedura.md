# Procedura Data Breach

Versione: 1.0
Ultimo aggiornamento: [YYYY-MM-DD]

## 1. Identificazione incidente

- Fonte: monitoraggio, segnalazione utente, alert sicurezza
- Aprire ticket incidente con timestamp e owner
- Classificare: confidenzialita / integrita / disponibilita

## 2. Contenimento immediato (entro poche ore)

- Isolare sistemi o account compromessi
- Revocare sessioni/token
- Bloccare endpoint/integrazioni coinvolte se necessario

## 3. Valutazione impatto

- Dati personali coinvolti
- Numero interessati stimato
- Probabilita e gravita del rischio per diritti/liberta

## 4. Notifica

- Se rischio per interessati: notifica al Garante entro 72 ore
- Se rischio elevato: informare anche gli interessati senza ritardo

## 5. Evidenze minime da registrare

- Data/ora rilevazione
- Sistemi coinvolti
- Dati coinvolti
- Azioni eseguite
- Decisione su notifica e motivazione

## 6. Chiusura e prevenzione

- Root cause analysis
- Piano di remediation con owner e scadenze
- Verifica efficacia post-fix
